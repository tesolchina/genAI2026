一、端到端工作流示意

1. 你描述任务 → 2. 规划模式（出计划/需求规格） → 3. 你批准计划 → 4. Contractor 自主执行 → 5. 多 Agent 审查（proofreader / slide-auditor / pedagogy / R / critic-fixer…） → 6. 修复并重审（至多 5 轮） → 7. 质量打分（0–100） → 8. 通过门控（≥80 提交，≥90 PR）才交付
「你不是给它一个个小指令，而是描述你要什么结果，然后它自己完成从规划到实施到质检的全过程。」
— wechatPost.md：Sant'Anna 建了一套什么系统
「每个文件最终会得到一个 0 到 100 的质量分数，80 分以下不允许提交到版本库。」
— wechatPost.md：同上

概念说明来源

三、在 Cursor 中如何复现工作流

本仓库在 claude-code-my-workflow/.cursor/ 下已提供 Cursor 专用技能与规则，无需安装额外插件，打开项目即可复现规划优先、多维度审查、质量门控与 MEMORY。

3.1 步骤

  1. 打开项目根目录：在 Cursor 中打开 courses/econometrics/claude-code-my-workflow/（即包含 CLAUDE.md 和 .cursor/ 的文件夹）。Cursor 会自动加载 .cursor/rules/；技能在 .cursor/skills/ 下可被 AI 根据描述触发。
  2. 依赖的文件：
    • 规则：.cursor/rules/*.mdc（plan-first-workflow、quality-gates、single-source-of-truth、session-logging、replication-protocol、exploration-folder、memory-learn）
    • 技能：.cursor/skills/*/SKILL.md（proofread、visual-audit、pedagogy-review、review-r、qa-quarto、slide-excellence、memory-and-learn）
    • 说明：.cursor/README.md、CLAUDE.md
  3. 首次适配：对 AI 说「请读 CLAUDE.md 和 .cursor/README.md，然后根据我当前项目更新 CLAUDE.md 里的占位符」。或粘贴下面 3.2 的提示词。
  4. 日常使用：用自然语言即可触发技能，例如：「对 Slides/Lecture01.tex 做 proofread」「对 Quarto/lecture1.qmd 做 slide-excellence」「审查 scripts/R 下所有 R 脚本」「先读 MEMORY.md 再开始」「对 Lecture1 跑一遍 qa-quarto」。

3.2 首次适配项目的提示词（可粘贴到 Cursor）

将 [PROJECT NAME] 和描述替换成你的项目（基于上游 Quick Start）：

I am starting to work on [PROJECT NAME] in this repo. [Describe your project in 2–3 sentences.]

I want our collaboration to be structured, precise, and rigorous. When creating visuals, everything must be polished and publication-ready.

Please read CLAUDE.md and .cursor/README.md, then update CLAUDE.md to fit my project (fill placeholders, adjust rules if needed). Use plan-first for non-trivial tasks: draft a plan, save to quality_reports/plans/, get my approval, then implement. When I correct you, append [LEARN:category] wrong → right to MEMORY.md.

Enter plan mode and start by adapting the workflow configuration for this project.

3.3 日常任务示例

  • 「把第三讲的 Beamer 转换成 Quarto，并跑一遍 slide-excellence。」
  • 「在 explorations/ 下新建子项目，试一个新工具变量设定，满足探索文件夹 60 分门槛。」
  • 「检查 scripts/ 下所有 R 脚本的复现协议：先复现基准再扩展。」

四、在 VS Code 中如何复现工作流

在 VS Code 中复现需使用 Claude Code 插件（或 CLI），依赖模板内的 .claude/ 配置，而非 .cursor/。

4.1 步骤

  1. 安装 Claude Code：CLI npm install -g @anthropic-ai/claude-code，或在 VS Code 中安装 Claude Code 扩展。见 Claude Code 文档 与上游 workflow guide – Setup。
  2. 打开项目根目录：在 VS Code 中打开 courses/econometrics/claude-code-my-workflow/（与 Cursor 相同根目录）。
  3. 依赖的文件：
    • .claude/agents/（proofreader、slide-auditor、pedagogy-reviewer、r-reviewer、quarto-critic、quarto-fixer 等）
    • .claude/skills/（/proofread、/visual-audit、/pedagogy-review、/review-r、/slide-excellence、/qa-quarto 等）
    • .claude/rules/（plan-first-workflow、quality-gates、single-source-of-truth、replication-protocol 等）
    • .claude/hooks/（pre-compact、session log 等；由 Claude Code 在适当时机调用）
    • CLAUDE.md、MEMORY.md
  4. 首次会话：在 Claude Code 面板中粘贴与 3.2 类似的提示词（或上游 Quick Start 的完整提示），让 Claude 读配置并适配项目。
  5. 日常使用：通过斜杠命令调用技能，如 /proofread Slides/Lecture01.tex、/slide-excellence Quarto/lecture1.qmd、/qa-quarto Lecture1。规划与 contractor 模式由 .claude/rules 驱动。

4.2 与 CLI 的差异

公众号 wechatPost.md 留言区作者回复：插件曾功能残缺，后已补部分功能；全部功能（如 agent teams）仍以 CLI 为准。MCP、hooks、skills 在插件中的支持请以当前 Claude Code 文档为准。

五、能做到什么程度（Cursor vs VS Code vs CLI）

Cursor 复现：本仓库已在 claude-code-my-workflow/.cursor/ 下提供 Cursor 版技能与规则，用于在 Cursor 内复现工作流。见 .cursor/README.md。

能力Cursor（.cursor/skills + rules）VS Code + Claude Code 插件Claude Code CLI
读写项目文件、执行命令✅ 支持✅ 支持✅ 支持
遵循 CLAUDE.md / .claude/rules✅ 项目内 .cursor/rules 等效✅ 同左✅ 原生支持
规划优先、Contractor 模式✅ plan-first-workflow 等规则✅ 同左✅ 原生支持
多 Agent 审查（proofreader、slide-auditor 等）✅ 单 agent 按 .cursor/skills 依次执行（proofread、visual-audit、pedagogy-review、review-r、slide-excellence、qa-quarto）⚠️ 视插件对 skills 的支持✅ 完整 skills/agents
Hooks、PreCompact、MEMORY.md✅ MEMORY.md 由 memory-learn 规则 + memory-and-learn 技能实现；Hooks 可手动或让 AI 执行 .claude/hooks 脚本⚠️ 依赖插件✅ 完整
Agent teams 等最新功能❌ 无多 agent 进程❌ 作者称插件尚未有✅ 有

结论：在 Cursor 中通过 .cursor/skills 与 .cursor/rules 已可复现「规划 → 实施 → 多维度审查（报告）→ 质量门控 → MEMORY 与探索文件夹」；多 agent 以单 agent 多技能顺序执行实现。Agent teams 等仍为 CLI 独有。
