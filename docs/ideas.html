Claude Code 学术工作流：端到端示意与 Cursor / VS Code 使用说明

本文档基于 pedrohcgs/claude-code-my-workflow 与公众号文章 wechatPost.md，说明工作流全流程、在 Cursor 与 VS Code 中的用法及可复现程度。

一、端到端工作流示意

1. 你描述任务 → 2. 规划模式（出计划/需求规格） → 3. 你批准计划 → 4. Contractor 自主执行 → 5. 多 Agent 审查（proofreader / slide-auditor / pedagogy / R / critic-fixer…） → 6. 修复并重审（至多 5 轮） → 7. 质量打分（0–100） → 8. 通过门控（≥80 提交，≥90 PR）才交付
「你不是给它一个个小指令，而是描述你要什么结果，然后它自己完成从规划到实施到质检的全过程。」
— wechatPost.md：Sant'Anna 建了一套什么系统
「每个文件最终会得到一个 0 到 100 的质量分数，80 分以下不允许提交到版本库。」
— wechatPost.md：同上

三、在 Cursor 中复现的步骤与提示词

3.1 准备

  • 本仓库中已包含克隆的模板：courses/econometrics/claude-code-my-workflow，或自行 fork 并 clone：https://github.com/pedrohcgs/claude-code-my-workflow。
  • 在 Cursor 中打开该克隆（或你 fork 的）项目根目录，确保 AI 能访问 CLAUDE.md、.claude/ 下的 rules、skills、agents。

3.2 首次适配项目的提示词（可粘贴到 Cursor AI）

以下基于上游 Quick Start 与 workflow-guide 首会话，将 [PROJECT NAME] 和描述替换成你的项目：

I am starting to work on [PROJECT NAME] in this repo. [Describe your project in 2–3 sentences — what you're building, who it's for, what tools you use.]

I want our collaboration to be structured, precise, and rigorous. When creating visuals, everything must be polished and publication-ready.

I've set up the Claude Code academic workflow (forked from pedrohcgs/claude-code-my-workflow). The configuration files are already in this repo. Please read them, understand the workflow, and then update all configuration files to fit my project — fill in placeholders in CLAUDE.md, adjust rules if needed, and propose any customizations specific to my use case.

After that, use the plan-first workflow for all non-trivial tasks. Once I approve a plan, switch to contractor mode — coordinate everything autonomously and only come back to me when there's ambiguity or a decision to make.

Enter plan mode and start by adapting the workflow configuration for this project.

3.3 日常任务示例

  • 「把第三讲的 Beamer 幻灯片转换成 Quarto 网页格式，并跑一遍质量审查。」
  • 「在 explorations/ 下新建一个子项目，试一个新的工具变量设定，满足 60 分门槛即可。」
  • 「检查 scripts/ 下所有 R 脚本的复现协议：先复现基准结果再扩展。」

在 Cursor 中，通过项目内的 .claude/rules 与 CLAUDE.md 约束行为，可实现“先规划、后执行、多轮审查、质量门控”的复现。

四、在 VS Code 中的用法与局限

  • VS Code + Claude Code 插件：打开同一项目，使用 Claude Code 面板，流程与 CLI 类似（见上游 full guide – Setup）。
  • 与 CLI 的差异（公众号留言区作者回复，wechatPost.md）：「三四个月前的时候 cc 插件是有残缺的，但后来补上了一些功能。但要所有功能的话肯定还是 cli，比如最新的 agent teams 的功能插件肯定还没有。」
  • 因此：MCP、hooks、skills 等在插件中可用性以当前 Claude Code 文档为准；若需 agent teams 等最新能力，建议用 CLI。

五、能做到什么程度（Cursor vs VS Code vs CLI）

能力Cursor（本机 AI + 项目规则）VS Code + Claude Code 插件Claude Code CLI
读写项目文件、执行命令✅ 支持✅ 支持✅ 支持
遵循 CLAUDE.md / .claude/rules✅ 需在项目内配置并让 AI 读取✅ 同左✅ 原生支持
规划优先、Contractor 模式✅ 通过规则与提示词实现✅ 同左✅ 原生支持
多 Agent 审查（proofreader、slide-auditor 等）⚠️ 由你或 AI 按技能描述调用，非自动多 agent⚠️ 同左，视插件对 skills 的支持✅ 完整 skills/agents
Hooks、PreCompact、MEMORY.md⚠️ 行为依赖 Cursor 是否注入这些机制⚠️ 依赖插件✅ 完整
Agent teams 等最新功能❌ 无❌ 作者称插件尚未有✅ 有

结论：在 Cursor 与 VS Code 中可复现「规划 → 实施 → 人工或半自动审查 → 质量意识」的主流程；要完全复现 Sant'Anna 的 10 agents、21 skills、7 hooks，仍以 Claude Code CLI 或插件后续更新为准。
