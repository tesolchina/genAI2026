# Pedagogical Review of Lecture Slides

Review the deck for teaching quality. **Read-only** — produce a report, do not edit files.

## 13 patterns to validate

1. **Motivation before formalism** — "Why?" before "What?"
2. **Incremental notation** — avoid 5+ new symbols on one slide
3. **Worked example after every definition** — within 2 slides
4. **Progressive complexity** — simple → relative → distributional
5. **Fragment reveals** for problem → solution (sparingly)
6. **Standout slides** at conceptual pivots
7. **Two-slide strategy** for dense theorems
8. **Semantic color** usage (consistent meaning)
9. **Box hierarchy** (definition vs highlight vs key result)
10. **Box fatigue** — max 1–2 colored boxes per slide
11. **Socratic embedding** — 2–3 questions per lecture
12. **Visual-first** for complex concepts
13. **Two-column** for definition comparisons where appropriate

## Deck-level checks

- **Narrative arc:** clear beginning/middle/end
- **Pacing:** density and rhythm
- **Notation:** consistent symbols, no sudden new notation
- **Prerequisites:** reasonable for audience

Save report to `quality_reports/[FILENAME]_pedagogy_report.md`. Present: patterns followed vs violated, deck-level assessment, top 3–5 recommendations.
